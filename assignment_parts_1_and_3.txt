Part 1 - Orientation

Three concepts I learned and used in my current research/work experience are:

1. I learned the importance of keeping work organized into clear steps so a larger project does not become confusing. In my PETase structure-activity project, I separated the workflow into data preparation, embedding extraction, merging embeddings with activity labels, and model training/evaluation.

2. I learned that reproducibility is important when doing technical work. Instead of manually changing notebooks every time, I am working toward using scripts and configuration-style settings so the same pipeline can be run again on different datasets, embedding types, and model settings.

3. I learned that workplace communication and planning are important when using shared computing resources. Since embedding protein structures with models like ESM-2 can be too slow on my personal computer, I need to coordinate with mentors to access external compute resources, understand where scripts should be run, and make sure the outputs can be shared back into the project.
